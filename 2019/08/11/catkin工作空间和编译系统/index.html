catkin工作空间和编译系统

发表于 2019-08-11 | 分类于 ros机器人操作系统

catkin工作区间

  • ros工程框架层次最高的概念

  • catkin是ros定制的编译构建系统，是对CMake的扩展

  • 编译ros程序的

catkin工作空间

  • 组织和管理功能包的文件夹
  • 以catkin工具编译
  • 创建、克隆、修改功能包在这个文件里完成

catkin_make(命令)

  • 编译工作空间

首先，建立工作空间

然后，编译

cmake_minimum_required() #指定catkin最低版本
project() #指定软件包的名称
find_package() #指定编译时需要的依赖项
add_message_files()/add_service_files()/add_action_files()
#添加消息文件/服务文件/动作文件

generate_messages() #生成消息、服务、动作
catkin_package() #指定catkin信息给编译系统生成Cmake文件
add_library()/add_executable() #指定生成库文件、可执行文件
target_link_libraries() #指定可执行文件去链接哪个库
catkin_add_gtest() #添加测试单元
install() #生成可安装目标
```

代码

ros的可执行程序有两种来源

  1. 脚本（Linux的shell、python脚本）
  2. C++(头文件、源文件)
  3. 通常在package下的scripts放可执行脚本（包括.py、.sh）
  4. 在package下的include放c++的头文件（*.h）
  5. 在package下的src放c++的源文件(*.cpp),可能也有python文件

自定义通讯格式

  1. 消息（msg）
  2. 服务（srv）
  3. 动作（action）

比如建一个服务，那么就在package下建一个srv文件（放*.srv文件）

launch文件以及配置文件

  • launch文件（launch）
    1. 一个package可以有多个可执行文件，要想每个都运行起来，那么就将它们组织到launch文件中，launch文件就是帮助我们一次性运行多个可执行文件
    2. 在package下建一个launch文件（放*.launch）
  • 配置文件（yaml）
    1. 参数、设置之类的
    2. 一般都在package下config文件下（放*.yaml）
    3. 自由度比较高，什么内容，什么格式一般都由自己来定，常用的就是.yaml格式

常用指令(包管理的指令)

  1. rospack
    • 查找某个包的地址
    • $ rospack find package_name
    • 列出本地所有pkg
    • $ rospack list
  2. roscd(相当于Linux下的cd)
    • 跳转到某个pkg路径下
    • $ roscd package_name
  3. rosls(相当于Linux下的ls)
    • 列举某个pkg下的文件信息
    • $ rosls packge_name
  4. rosed(相当于Linux下的vi(m))
    • 编辑pkg中的文件
    • $ roscd package_name file_name
  5. catkin_create_pkg
    • 创建一个pkg
    • catkin_create_pkg <pkg_name> [deps#包所需的依赖]
  6. rosdep
    • 安装某个pkg所需的依赖
    • 适用场合，如从GitHub上下载的包
    • $ rosdep install [pkg_name]

e.g
- $ mkdir -p catkin_ws/src
- $ cd catkin_ws/
- $ catkin_make
- cd src
- $ catkin_create_pkg test1
- catkin_create_pkg test2 roscpp#c++编程的API rospy std_msgs#通讯 nav_msgs#导航
