catkin工作空间探索及Metapackage

发表于 2019-08-12 | 分类于 ros机器人操作系统

代码
rosdep install --from-paths src --ignore-src --rosdistro melodic -y
作用是添加代码包所需要的依赖，但rosdep怎么知道要添加什么依赖包呢，其实是定义在package下的package.xml文件里面的build_depend和run_depend这两个标签，告诉rosdep编译和运行时一定有这些依赖项，rosdep知道要把这些依赖项装好。

rospack list | grep catkin_wsros包在catkin_ws的管道列表

Metapackage

  • Linux软件包管理的概念
  • 虚包，没有什么实质性内容
  • 依赖很多其他软件包
  • 如 在sudo apt-get install ros-melodic-desktop-full其实就是安装了一个Metapackage，包含ros的一些组件，ros的编译系统，ros的基本文件系统，一些指令等等。不需要安装这些所有的软件包，只需安装这一个就够了。所以metapackage就是将它依赖的软件包组合起来。stack叫作软件包集，最后废弃了。
  • 作用：就是安装时更方便，将功能相似的软件包给组织起来
Metapackage名称 描述
navigation 导航相关的功能包集
moveit 运动规划相关的（主要是机械臂）功能包集
image_pipeline 图像获取、处理相关的功能包集
vision_opencv ROS与OpenCV交互的功能包集
turtlebot Turtlebot机器人相关的功能包集
pr2_robot pr2机器人驱动功能包集
