Client Library(客户端库)

  • 概念类似于API
    区别：
    client library的层级要更高，它是在API的基础上又做了一层封装，对开发者更加友好
  • 提供ros编程的库,就是一个接口，它把底层的标准流程给封装好，方便程序员去写代码
    例如：建立node，发布消息，调用服务…

ros提供的client library

  1. roscpp c++ 执行效率高，适合做图像、slim这方面的工作，代码写起来麻烦
  2. rospy python 开发效率高，小工具，小node都可以用rospy来写。
    一些ros命令行工具，像rosrun、rostopic都是用rospy来实现的
    这些命令都是一个个python脚本，最底层就是rospy来实现的
  3. roslisp

roscpp

  • ros提供的用c++和topic、service、param、timer交互的一个接口
  • 官方文档中

ros::init（）初始化函数，在写c++程序时，必须要调用的一个函数
ros::NodeHandle:最重要的一个部分，用来创建topic、service和调用这些公共接口
ros::master：通过它获取和master交互来查询系统的信息
ros::this_node:用来查询进程的函数
ros::service:查询服务的函数
ros::param:查询参数服务器
ros::names:名称管理用到的功能

ros::NodeHandle Class(类)

  • c++里面的类有自己的成员变量和成员函数
  • 怎么用
    创建一个话题的publisher
  1. 首先声明一个ros::NodeHandle nh；(对象)
  2. 用这个函数的话，就ros::Publisher pwb = nh.advertise(…);//有两个参数，第一个参数是写topic的名称，指定这个publisher往哪个topic去发布消息，第二个王消息队列上发布消息的长度
  3. 调用完了advertise函数返回值就是一个ros::Publisher赋给一个Publisher的对象
  4. 发布消息，就直接用pwb.publish(msg);
    形如

参数有topic名称，队列的长度，还有一个回调函数的名称（函数指针）当subscribe到一个msg时就会由回调函数来处理这个msg
