端口重映射及蜂鸣器操作方法

端口重映射

蜂鸣器原理图如下：

三极管作用：放大电流
10k电阻作用：限流
二极管作用：与蜂鸣器构成放电回路，防止烧坏三极管
为什么要端口重映射？
因为蜂鸣器的IO口PB4原本的功能是JTAG调试，而非普通IO口
重映射方法：

  1. 开启AFIO复用时钟RCC_APB2Periph_AFIO(所有重映射功能都在AFIO这个时钟管理下面)
    RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOB|RCC_APB2Periph_AFIO, ENABLE);
  2. 调用GPIO_PinRemapConfig配置管脚重映射
    GPIO_PinRemapConfig(GPIO_Remap_SWJ_NoJTRST, ENABLE);
  3. 注意映射后端口的功能（本实验中重映射后作为普通IO口使用）
    库函数中有例程可以参考GPIO/JTAG_Remap

Stm32时钟简介

  • RCC 复位和时钟总线
  • AHB 系统总线（高阶外设总线）
  1. APB1 定时器、串口、低速
  2. APB2 GPIO、高速
    HSE:高速外部时钟
    HSI:高速内部时钟
    LSI:低速内部时钟
    LSE:低速外部时钟
  3. 了解APB1(低速)和APB2(高速)外设
  4. 学会看STM32时钟框图
  5. 学会用库函数开启外设时钟
  6. 学会STM32 IO配置方法与应用场合

定义初始化结构体

GPIO_InitTypeDef GPIO_InitStructure
定义结构体变量类型 结构体名称
GPIO_Mode
枚举变量构成
GPIO的八种工作模式

  1. GPIO_Mode_AIN
    模拟输入 ADC采样
  2. GPIO_Mode_IPD
    下拉输入 IO口不接的话，默认为低电平
  3. GPIO_Mode_IPU
    上拉输入 IO口不接的话，默认为高电平
  4. GPIO_Mode_IN_FLOATING
    浮空输入 IO口不接的话，状态不定，呈高阻态
  5. GPIO_Mode_Out_OD
    开漏输出，IO输出0接GND，IO输出1，悬空，需要外接上拉电阻，才能实现输出高电平。当输出为1时，IO口的状态由上拉电阻拉高电平，但由于是开漏输出模式，这样IO口也就可以由外部电路改变为低电平或不变。可以读IO输入电平变化，实现C51的IO双向功能，适合于做电流型的驱动,其吸收电流的能力相对强(一般20ma以内)。
  6. GPIO_Mode_Out_PP
    推挽输出，可以输出高电平、低电平（提高电路负载能力，开关速度）
  7. GPIO_Mode_AF_OD
    复用开漏输出，片内外设功能
  8. GPIO_Mode_AF_PP
    复用推挽输出，片内外设功能

IO口配置步骤

  1. 定义一个IO口初始化结构体 GPIO_InitTypeDef GPIO_InitStructure;

  2. 使能相应的IO口时钟 RCC_APB2PeriphClockCmd(RCC_APB2Periph_GPIOC, ENABLE);

  3. 结构体赋值

  4. 调用初始化函数 GPIO_Init(GPIOC, &GPIO_InitStructure);

LED锁存器控制

  • D0-D7对应的GPIO引脚为PC8-PC15
  • LE锁存器控制引脚为PD2

LED为共阳模式

  • GPIO_ResetBits(GPIOC,led_ctrl<<8); 把对应引脚清0，不影响其他位
    例如，将led_ctrl赋值为0x01，则GPIOC的第八个引脚被赋值为0。
  • GPIO_SetBits(GPIOD,GPIO_Pin_2); 把对应的引脚置1，不影响其他位
  • 编写LED控制代码的方法
    例如，使LED亮，设置该LED对应的GPIO口的状态，再打开锁存器使PC口的状态做出的改变输出到LED口。之后，关闭锁存器使该状态保持，并且保持的状态不会改变。
    GPIO_ResetBits(GPIOC, led_ctrl<<8);
  GPIO_SetBits(GPIOD, GPIO_Pin_2);
  GPIO_ResetBits(GPIOD, GPIO_Pin_2);

    调试时能看的变量：全局变量与未被优化的局部变量

URDF

  • Unified Robot Description Format统一机器人描述格式
  • 这个文件制定了一个规范，用来描述这个机器人是这样的一个结构
  • tf、可视化、仿真的功能都需要urdf的参与
  • ros里面有个package为urdf本质是一个解析器，用来解析urdf格式的文件
  • urdf格式中定义了两个部分
  1. link 就是指部件、零件，每一个link都有三个轴，分别代表x,y,z
  2. joint 就是连接两个link之间的关节，表示两个link之间的连接关系，和tf是对应的

ros::spin()
每来一个消息就会把消息放在队列里，何时去处理呢？
并不是来一个就会处理一个，必须调用ros::spin()函数，这个ros::spin()会反复查看回调函数的队列是否有东西
如果有就在这个队列的东西清空，把里面的每一个msg进行处理，如果队列是空的，就不会调用这个回调函数来处理
ros::spin()是反复调用当前可触发的回调函数，执行到这一句它就会反复查看有无可执行的回调函数，所以是阻塞的
ros::spinOnce()仅查看一次有无可处理的队列，有的话就处理，没有的话就继续向下执行了，因此它不是阻塞的函数
创建Subscriber来接收某个topic上的消息，然后用回调函数来处理对应的消息，回调函数不会自动触发
需要调用ros::spin()或ros::spinOnce()来触发
