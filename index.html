Float32.msg
ros自带的Float32类型是一个结构体，但里面只有一个内容就是data，data才是真正数据的地方
ros::spin()
每来一个消息就会把消息放在队列里，何时去处理呢？
并不是来一个就会处理一个，必须调用ros::spin()函数，这个ros::spin()会反复查看回调函数的队列是否有东西
如果有就在这个队列的东西清空，把里面的每一个msg进行处理，如果队列是空的，就不会调用这个回调函数来处理
ros::spin()是反复调用当前可触发的回调函数，执行到这一句它就会反复查看有无可执行的回调函数，所以是阻塞的
ros::spinOnce()仅查看一次有无可处理的队列，有的话就处理，没有的话就继续向下执行了，因此它不是阻塞的函数
创建Subscriber来接收某个topic上的消息，然后用回调函数来处理对应的消息，回调函数不会自动触发
需要调用ros::spin()或ros::spinOnce()来触发

发表于 2019-08-13 | 分类于 ros机器人操作系统

parameter server（参数服务器）

  • 用roscore命令时已经把参数服务器启动好了
  • 有别于ros里面同步通信和异步通信的通信方式
  • 更加静态
  • 存储着各种参数和配置的字典
  • 可用命令行，launch文件和node（API（在ros的源代码里））读写（维护方便）
  • 字典就是一个个的键值对，字典里面的key不能重复，都是唯一的，每一个key对应着一个value
  • 字典就是一个映射关系，在实际的项目里，我们往往把一些不常改变的参数、配置放到参数服务器里即字典里
  • 这样对它进行读写都非常方便
    字典形如
    Key|/rosdistro|/rosversion|/use_sim_time|…
  • |-|-|-|-
    Value|’kinetic’|’1.12.7’|true|…

常用命令rosparam

  1. 列出当前所有参数
    $ rosparam list
  2. 显示某个参数的值
    $ rosparam get param_key
  3. 设置某个参数的值
    $ rosparam set param_key param_value
  4. 保存参数到文件
    $ rosparam dump file_name
  5. 从文件读取参数
    $ rosparam load file_name#从文件读取参数到字典里
  6. 删除参数参数
    $ rosparam delete param_key
    注：dump或者load的文件要遵从YAML格式
