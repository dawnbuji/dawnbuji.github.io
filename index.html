URDF

  • Unified Robot Description Format统一机器人描述格式
  • 这个文件制定了一个规范，用来描述这个机器人是这样的一个结构
  • tf、可视化、仿真的功能都需要urdf的参与
  • ros里面有个package为urdf本质是一个解析器，用来解析urdf格式的文件
  • urdf格式中定义了两个部分
  1. link 就是指部件、零件，每一个link都有三个轴，分别代表x,y,z
  2. joint 就是连接两个link之间的关节，表示两个link之间的连接关系，和tf是对应的

ros::spin()
每来一个消息就会把消息放在队列里，何时去处理呢？
并不是来一个就会处理一个，必须调用ros::spin()函数，这个ros::spin()会反复查看回调函数的队列是否有东西
如果有就在这个队列的东西清空，把里面的每一个msg进行处理，如果队列是空的，就不会调用这个回调函数来处理
ros::spin()是反复调用当前可触发的回调函数，执行到这一句它就会反复查看有无可执行的回调函数，所以是阻塞的
ros::spinOnce()仅查看一次有无可处理的队列，有的话就处理，没有的话就继续向下执行了，因此它不是阻塞的函数
创建Subscriber来接收某个topic上的消息，然后用回调函数来处理对应的消息，回调函数不会自动触发
需要调用ros::spin()或ros::spinOnce()来触发

ros常用工具

发表于 2019-08-13 | 分类于 ros机器人操作系统

ros常用工具

  1. 仿真：gazebo
  2. 调试、可视化：rviz、rqt
  3. 命令行工具：rostopic、rosbag…
  4. 专用工具：Moveit!

Gazebo

  • 与ros都是OSRF开源机器人组织开发的
  • 机器人仿真工具 还有像V-Rep、carsim（无人驾驶领域）之类的仿真工具包
  • ODE物理引擎 gazebo可以兼容很多物理引擎，主要还是ODE这个开源的物理引擎，其他的像Bullet也行
  • 用于动力学、导航、感知等任务的模拟 像无人驾驶、强度学习很多都会在模拟器中进行训练、进行模拟

RViz

  • 全称 The Robot Visualization tool 机器人可视化工具
  • 把一些抽象的、复杂的传感器信息用图像的方式呈现给我们
  • 方便监控和调试
  • 点云：就是一个个小方格，每个点不仅有颜色（rgb）信息，而且有位置（x轴、y轴、z轴）信息

rqt

  • 全称rosqt，基于qt开发的可视化工具
  • 常用rqt_graph,rqt_plot,rqt_console
  • 扩展性好，跨平台的性能也很好，
  • 与Rviz相比，显示的东西更高级一些
  • rqt_graph:显示通信架构 当前有些node在运行，有哪些topic，消息的流向从哪里到哪里，可以反映系统的全貌
  • rqt_plot:绘制全线，它把一些参数，尤其是动态参数的曲线给绘制出来，用于看原始数据，比如看速度，直观地把曲线绘制出来
  • rqt_console：查看日志，带一些过滤器，把日志分成了不同等级

    上图是一个当前运行的计算图，它把整个系统的node和消息的流向都画出来了
    其中圆圈里面的是节点，箭头上的是topic

rosbag

  • ros里数据记录、数据回放系统
  • ros命令行工具
  • 记录和回放数据流
  • 其中的record功能就可以把指定的topic记录下来，设置录制的长度、消息的个数
  • 这些消息都会存在.bag文件里，可以回放play
    记录和回放的原理:
    记录的话，就是创建了一个node，把你需要的订阅的消息都记录下来，然后打上时间戳

常用命令

  1. 记录某些topic到bag中
    $ rosbag record <topic-names>
    一般录制到bag文件名为类似于2019-8-14 09:36:42.bag文件中
  2. 记录所有topic到bag中
    $ rosbag record -a
  3. 回放bag
    $ rosbag play <bag-files>
    回放时，即开了新的节点，所以在命令行先输入roscore

action

  • ros中的一种通信方式
  • 相当于一个功能更强大的service
  • cline可以cancel请求，server可以feedback状态反馈，实时进度跟进
  • 类似于service，带有状态反馈的通信方式
    通信结构如下
  1. goal 给server的请求，希望这个动作能实现的目标
  2. cancel 任务执行中间需要cancel掉这个任务
  3. status sever现在所处的一个状态
  4. result 动作处理完后会返回一个结果给client
  5. feedback 实时状态会反馈给client
  • 通常用于长时间、可抢占的任务
