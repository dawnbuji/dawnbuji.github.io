ros常用工具

发表于 2019-08-13 | 分类于 ros机器人操作系统

ros常用工具

  1. 仿真：gazebo
  2. 调试、可视化：rviz、rqt
  3. 命令行工具：rostopic、rosbag…
  4. 专用工具：Moveit!

Gazebo

  • 与ros都是OSRF开源机器人组织开发的
  • 机器人仿真工具 还有像V-Rep、carsim（无人驾驶领域）之类的仿真工具包
  • ODE物理引擎 gazebo可以兼容很多物理引擎，主要还是ODE这个开源的物理引擎，其他的像Bullet也行
  • 用于动力学、导航、感知等任务的模拟 像无人驾驶、强度学习很多都会在模拟器中进行训练、进行模拟

RViz

  • 全称 The Robot Visualization tool 机器人可视化工具
  • 把一些抽象的、复杂的传感器信息用图像的方式呈现给我们
  • 方便监控和调试
  • 点云：就是一个个小方格，每个点不仅有颜色（rgb）信息，而且有位置（x轴、y轴、z轴）信息

rqt

  • 全称rosqt，基于qt开发的可视化工具
  • 常用rqt_graph,rqt_plot,rqt_console
  • 扩展性好，跨平台的性能也很好，
  • 与Rviz相比，显示的东西更高级一些
  • rqt_graph:显示通信架构 当前有些node在运行，有哪些topic，消息的流向从哪里到哪里，可以反映系统的全貌
  • rqt_plot:绘制全线，它把一些参数，尤其是动态参数的曲线给绘制出来，用于看原始数据，比如看速度，直观地把曲线绘制出来
  • rqt_console：查看日志，带一些过滤器，把日志分成了不同等级

rosbag

  • ros里数据记录、数据回放系统
  • ros命令行工具
  • 记录和回放数据流
  • 其中的record功能就可以把指定的topic记录下来，设置录制的长度、消息的个数
  • 这些消息都会存在.bag文件里，可以回放play
    记录和回放的原理:
    记录的话，就是创建了一个node，把你需要的订阅的消息都记录下来，然后打上时间戳

常用命令

  1. 记录某些topic到bag中
    $ rosbag record <topic-names>
    一般录制到bag文件名为类似于2019-8-14 09:36:42.bag文件中
  2. 记录所有topic到bag中
    $ rosbag record -a
  3. 回放bag
    $ rosbag play <bag-files>
    回放时，即开了新的节点，所以在命令行先输入roscore

parameter server（参数服务器）

  • 用roscore命令时已经把参数服务器启动好了
  • 有别于ros里面同步通信和异步通信的通信方式
  • 更加静态
  • 存储着各种参数和配置的字典
  • 可用命令行，launch文件和node（API（在ros的源代码里））读写（维护方便）
  • 字典就是一个个的键值对，字典里面的key不能重复，都是唯一的，每一个key对应着一个value
  • 字典就是一个映射关系，在实际的项目里，我们往往把一些不常改变的参数、配置放到参数服务器里即字典里
  • 这样对它进行读写都非常方便
    字典形如
    Key|/rosdistro|/rosversion|/use_sim_time|…
  • |-|-|-|-
    Value|’kinetic’|’1.12.7’|true|…

常用命令rosparam

  1. 列出当前所有参数
    $ rosparam list
  2. 显示某个参数的值
    $ rosparam get param_key

    $ rosparam set param_key param_value
  4. 保存参数到文件
    $ rosparam dump file_name
  5. 从文件读取参数
    $ rosparam load file_name#从文件读取参数到字典里
  6. 删除参数参数
    $ rosparam delete param_key

Service与srv

发表于 2019-08-13 | 分类于 ros机器人操作系统

Service

  • ros中的同步通信方式
    同步通信是client发送一个请求会一直在原地等着，会阻塞，直到服务器端返回来结果，node A才会继续执行
  • node间可以通过request-reply方式通信（请求和响应）
    node A（client）客服端 node B（Server）服务器 node B会给外界提供一个/Service（服务），别的node来请求node B来调用这个服务，即node A会把Service需要的参数传给Service也就是请求它，node B收到请求就会做处理，处理完了就会回复给node A

Topic VS Service

对象 Topic Service
通信方式 异步通信 同步通信
实现原理 TCP/IP TCP/IP
通信模型 Publish-Subscribe Request-Reply
publish-subscribe（多对多） client-server（多对一）
接收者收到数据会回调（callback） 远程过程调用（RPC）服务器端的服务
应用场景 连续、高频的数据发布 偶尔调用的功能/具体的任务
举例 激光雷达、里程计发布数据 开关传感器、拍照、逆解计算

注：

  1. 回调函数来做一个处理
  2. rpc就是在一个进程（node）里面去调用另一个进程（node）里面的函数
  3. 远程过程调用是和本地函数调用这个概念是相对的

常用命令rosservice

  1. 列出当前所有活跃的service
    $ rosservice list
  2. 显示某个service的属性信息
    $ rosservice info service_name
  3. 调用某个service
    $ rosservice call service_name args(希望传入的参数)

srv

  • service的简写
  • service通信的数据格式
  • 定义在*.srv文件中
  • *.srv文件一定要放在srv这个文件下
  • 形如
    my_pkg/srv/DetectHuman.srv

my_pkg/msg/HumanPose.msg #msg一种自定义的数据类型

my_pkg/msg/JointPos.msg #定义JointPos[]数组

定义了srv或者msg，要修改package.xml和CMakeList.txt，具体就是添加一些条目，指定了srv和msg编译运行时所需要的依赖
只有自定义了srv或者msg就要把这几条添进去

常用命令rossrv

  1. 列出系统上所有的srv
    $ rossrv list
  2. 显示某个srv内容
    $ rossrv show srv_name

Topic和Msg

发表于 2019-08-12 | 分类于 ros机器人操作系统

通信方式

  1. Topic(主题)
  2. Service(服务)
  3. Parameter Service(参数服务器)
  4. Actionlib(动作库)

Topic

  • ros中用的最多的一种通信方式
  • ros中的异步通信方式
  • Node间通过publish-subscribe机制通信
    node A向Topic发送消息，node B向Topic订阅消息，topic就像是一个频道，node A向这个频道发，node B向这个频道收听
  • 是一个字符串string

    什么是异步？

    例：node1（camera）->publish(发布)->/camera_rgb、/camera_depth(Topic)
    这个node只管发布topic，不管有没有人接收和处理
    node2(image process)->sub（订阅）->/camera_rgb、/camera_depth(Topic)
    这个node只管消息来了 接收、处理 不管谁发布的
    对于发布者来说，当它发布了这个消息，它不用等待这个消息返回的状态
    对于接受者来说，有消息到了就去处理，没有消息就可以做别的事情
    这就是一种异步的通信方式
    node3（display）:也可以同时订阅/camera_rgb等topic
  1. topic的通信方式是异步的，在发送的时候会调用这个publish（）这个方法，调用完这个函数，就可以立即返回了，不用等待发送之后的处理的结果
  2. topic可以有多个subscribers（接收者），同样地，一个topic也可以有多个publishers（发送者）

Topic相关命令

  1. 列出当前所有topic
    $ rostopic list
  2. 显示某个topic的属性信息
    $ rostopic info /topic_name
  3. 显示某个topic的内容
    $ rostopic echo /topic_name
  4. 向某个topic发布内容
    $ rostopic pub /topic_name ...

Message

  • 是topic内容的数据类型，topic的格式标准。message是一种数据格式，信息要遵守的规范。相当于类这个概念。
  • 定义在.msg文件中 `pkg/msg/.msg`
  • 基本的msg包括
    bool,int8,int16,int64(以及uint)
    float32,float64,sting
    time,duration,header
    可变长数组array[],固定长度数组array[C]
    message就相当于一个结构体或者说是类

Message相关命令

  1. 列出系统上所有msg
    $ rosmsg list
  2. 显示某个msg内容
    $ rosmsg show /msg_name

其中一些topic如/camera/rgb … 和/camera/depth …,即图像的色彩和深度，两者信息融合后就可以构成点云（pointcloud），这个数据，用于定位、建图、识别这些功能上。

Mater和Node

发表于 2019-08-12 | 分类于 ros机器人操作系统

ROS通信架构（计算图级）

  • 将程序跑起来会有一个怎样的网络结构，会有一个什么样的框架
  • personrobot2简称pr2
  • 怎样管理进程，它们之间怎样通信
  • ros的首要目标就是方便各个模块之间的管理和通信
  • 启动这些进程，需要一个管家管理这些要启动的进程-master（节点管理器）

master

  • master的作用有两个
    1. 每个node（节点）启动时都要向master注册
    2. 管理node（节点）之间的通信，node之间点对点的通信
  • 启动ros程序之前，第一步就是启动master
  • 启动ros master的方法：$ roscore
  • $roscore启动了以下三个东西
    1. master：节点管理器
    2. rosout：日志输出 node 告诉我们系统有什么error、有什么warning，把这些log记录下来
    3. parameter server：参数服务器 做一些参数的配置
  • 启动了mater之后就可以启动一些具体的进程了，即node（节点）

node (节点)

  • ros的进程，在ros中叫作节点
  • pkg里的可执行文件（c++编译的、.python、.shell）运行的实例。当我们运行这些可执行文件，它们被加载到了内存中就成为了一个node
  • 可执行文件是静态的，node是动态的，一般来说一个node就做某个特定的功能，就专注做某件事
  • node一般都是以功能来划分的，不同的功能放在不同的node里

node启动的方法-rosrun

启动一个node
$ rosrun [pkg_name] [node_name]

结束某个node
$ rosnode kill [node_name]

Metapackage

  • Linux软件包管理的概念
  • 虚包，没有什么实质性内容
  • 依赖很多其他软件包
  • 如 在sudo apt-get install ros-melodic-desktop-full其实就是安装了一个Metapackage，包含ros的一些组件，ros的编译系统，ros的基本文件系统，一些指令等等。不需要安装这些所有的软件包，只需安装这一个就够了。所以metapackage就是将它依赖的软件包组合起来。stack叫作软件包集，最后废弃了。
  • 作用：就是安装时更方便，将功能相似的软件包给组织起来

线性规划

发表于 2019-08-08 | 分类于 数学建模

线性规划问题

线性规划的实例与定义

  • 例1.1 某机床厂生产甲、乙两种机床，每台销售后的利润分别为4000元与3000元。生产甲机床需用A、B机器加工，加工时间分别为每台2h和1h；生产乙机床需用A、B、C三种机器加工，加工时间为每台各1h。若每天可用于加工的机器实数分别为A机器10h、B机器8h和C机器7h，问该厂应生产甲、乙机床各几台，才能使总利润最大？
    上述问题的数学模型：设该厂生产x_1台甲机床和x_2乙机床时总利润z最大，则x_1,x_2应满足
    max z = 4x_1 + 3x_2(1.1)
    s.t. 2x_1 + x_2 <= 10
    x_1+x_2 <= 8
    x_1,x_2 >= 0

式中：变量 x_1, x_2为决策变量

式（1.1）称为问题的目标函数，式（1.2）中的几个不等式是问题的约束条件，记为s.t.（即subject to）。由于上面的目标函数及约束条件均为线性函数，故称为线性规划问题。

总之，线性规划问题是在一组线性约束条件的限制下，求一线性目标函数最大或最小的问题。

在解决实际问题时，把问题归结成一个线性规划数学模型是很重要的一步，往往也是很困难的一步，模型建立得是否得当，直接影响到求解。而选择适当的决策变量，是建立有效模型的关键之一。

线性规划的MATLAB标准形式与软件求解

  • MATLAB中规定线性规划的标准形式，目标函数是求最小值，约束条件的不等号是小于等于号
  • MATLAB中求解线性规划的命令为
[x,fval] = linprog(f,A,b)
[x,fval] = linprog(f,A,b,Aeq,beq)
[x,fval] = linprog(f,A,b,Aeq,beq,lb,ub)
