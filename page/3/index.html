catkin工作空间和编译系统

发表于 2019-08-11 | 分类于 ros机器人操作系统

catkin工作区间

  • ros工程框架层次最高的概念

  • catkin是ros定制的编译构建系统，是对CMake的扩展

  • 编译ros程序的

catkin工作空间

  • 组织和管理功能包的文件夹
  • 以catkin工具编译
  • 创建、克隆、修改功能包在这个文件里完成

catkin_make(命令)

  • 编译工作空间

首先，建立工作空间

然后，编译

要在catkin_ws目录下catkin_make
记得编译完的catkin_ws刷新到环境变量里面去，系统可以知道我们生成的ros可执行文件放在哪里

  • src:package源代码，写代码的地方
  • build：cmake&catkin缓存和中间文件
  • devel：目标文件（头文件、动态静态的链接库和可执行文件）

src

package

  • 是catkin编译的基本单元，catkin系统会递归的查找每一个package
  • ros软件的基本组织形式，相当于ros的细胞
  • 一个package可以包含多个可执行文件（节点）
  • 包含至少CMakeLists.txt和package.xml两个文件夹

    CMakeLists.txt

  • 规定catkin编译的规则（告诉编译系统怎么package里面的代码，要编译哪些文件、需要哪些依赖、生成哪些可执行目标、怎么链接等等）
  • 例如：源文件、依赖项、目标文件
  • 编译出了什么问题一般跟这个CMakeList.txt这个文件和依赖包有关系

package.xml

  • 定义package的属性，相当于包的自我描述
  • 例如：包名、版本号、作者、依赖等
  • 模板

代码

ros的可执行程序有两种来源

  1. 脚本（Linux的shell、python脚本）
  2. C++(头文件、源文件)
  3. 通常在package下的scripts放可执行脚本（包括.py、.sh）
  4. 在package下的include放c++的头文件（*.h）
  5. 在package下的src放c++的源文件(*.cpp),可能也有python文件

自定义通讯格式

  1. 消息（msg）
  2. 服务（srv）
  3. 动作（action）

比如建一个服务，那么就在package下建一个srv文件（放*.srv文件）

launch文件以及配置文件

  • launch文件（launch）
    1. 一个package可以有多个可执行文件，要想每个都运行起来，那么就将它们组织到launch文件中，launch文件就是帮助我们一次性运行多个可执行文件
    2. 在package下建一个launch文件（放*.launch）
  • 配置文件（yaml）
    1. 参数、设置之类的
    2. 一般都在package下config文件下（放*.yaml）
    3. 自由度比较高，什么内容，什么格式一般都由自己来定，常用的就是.yaml格式

常用指令(包管理的指令)

  1. rospack
    • 查找某个包的地址
    • $ rospack find package_name
    • 列出本地所有pkg
    • $ rospack list
  2. roscd(相当于Linux下的cd)
    • 跳转到某个pkg路径下
    • $ roscd package_name
  3. rosls(相当于Linux下的ls)
    • 列举某个pkg下的文件信息
    • $ rosls packge_name
  4. rosed(相当于Linux下的vi(m))
    • 编辑pkg中的文件
    • $ roscd package_name file_name
  5. catkin_create_pkg
    • 创建一个pkg
    • catkin_create_pkg <pkg_name> [deps#包所需的依赖]
  6. rosdep
    • 安装某个pkg所需的依赖
    • 适用场合，如从GitHub上下载的包
    • $ rosdep install [pkg_name]

e.g
- $ mkdir -p catkin_ws/src
- $ cd catkin_ws/
- $ catkin_make
- cd src
- $ catkin_create_pkg test1
- catkin_create_pkg test2 roscpp#c++编程的API rospy std_msgs#通讯 nav_msgs#导航

线性规划

发表于 2019-08-08 | 分类于 数学建模

线性规划问题

线性规划的实例与定义

  • 例1.1 某机床厂生产甲、乙两种机床，每台销售后的利润分别为4000元与3000元。生产甲机床需用A、B机器加工，加工时间分别为每台2h和1h；生产乙机床需用A、B、C三种机器加工，加工时间为每台各1h。若每天可用于加工的机器实数分别为A机器10h、B机器8h和C机器7h，问该厂应生产甲、乙机床各几台，才能使总利润最大？
    上述问题的数学模型：设该厂生产x_1台甲机床和x_2乙机床时总利润z最大，则x_1,x_2应满足
    max z = 4x_1 + 3x_2(1.1)
    s.t. 2x_1 + x_2 <= 10
    x_1+x_2 <= 8
    x_1,x_2 >= 0

式中：变量 x_1, x_2为决策变量

式（1.1）称为问题的目标函数，式（1.2）中的几个不等式是问题的约束条件，记为s.t.（即subject to）。由于上面的目标函数及约束条件均为线性函数，故称为线性规划问题。

总之，线性规划问题是在一组线性约束条件的限制下，求一线性目标函数最大或最小的问题。

在解决实际问题时，把问题归结成一个线性规划数学模型是很重要的一步，往往也是很困难的一步，模型建立得是否得当，直接影响到求解。而选择适当的决策变量，是建立有效模型的关键之一。

线性规划的MATLAB标准形式与软件求解

  • MATLAB中规定线性规划的标准形式，目标函数是求最小值，约束条件的不等号是小于等于号
  • MATLAB中求解线性规划的命令为
[x,fval] = linprog(f,A,b)
[x,fval] = linprog(f,A,b,Aeq,beq)
[x,fval] = linprog(f,A,b,Aeq,beq,lb,ub)
